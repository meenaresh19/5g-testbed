================================================================================
                         MIGRATION HIGHLIGHTS
================================================================================

BEFORE (Docker Compose):
  └─ docker-compose.yml → docker-compose up
  └─ Hardcoded IPs (10.45.0.x)
  └─ Docker socket API (dockerode)
  └─ Manual scaling (edit + restart)
  └─ No RBAC security
  └─ Named volumes (basic storage)

AFTER (Kubernetes K3s):
  └─ Manifest files → kubectl apply -k
  └─ K8s DNS services (service-name.namespace.svc)
  └─ K8s client API (@kubernetes/client-node) + RBAC
  └─ Declarative scaling (kubectl scale)
  └─ Full RBAC (ServiceAccount + ClusterRole)
  └─ PersistentVolumeClaims (production patterns)

BENEFITS:
  ✅ Cloud-native infrastructure
  ✅ Scalable architecture
  ✅ Industry-standard security
  ✅ Multi-cloud portability
  ✅ Production-ready patterns

================================================================================
                            API ENDPOINTS
================================================================================

GET http://localhost:30080/api/status
  → Returns list of all NFs and their status

POST http://localhost:30080/api/nf/{id}/start
  → Scale NF deployment to 1 replica

POST http://localhost:30080/api/nf/{id}/stop
  → Scale NF deployment to 0 replicas

GET http://localhost:30080/api/nf/{id}/logs?lines=100
  → Get logs from NF pod

GET http://localhost:30080/api/nf/{id}/config
  → Get NF configuration

GET http://localhost:30080/api/metrics/query?query=...
  → Prometheus metric query

Examples:
  curl http://localhost:30080/api/status
  curl -X POST http://localhost:30080/api/nf/open5gs-smf/start
  curl http://localhost:30080/api/nf/open5gs-amf/logs?lines=50
